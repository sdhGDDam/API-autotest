## Запуск тестов

### Подготовка окружения

1. Склонируйте репозиторий
2. Установите расширение **Go** от Google (ms-vscode.go).
3. Откройте папку с проектом в VS Code.
4. Убедитесь, что все зависимости подтянуты: откройте терминал (`Ctrl + Shift + ~`) и выполните:
   ```bash
   go mod tidy

### Настройка переменных окружения

В корне проекта создайте файл **`.env.override`** (он имеет приоритет над `.env`) со следующим содержимым(к примеру):

```env
TEST_LOGIN=test@test.ru
TEST_PASSWORD=Testtest123
```

### Тестовые фотографии

Для работы позитивных тестов требуются три тестовых изображения. Они уже включены в репозиторий и лежат в папке **`testdata/`** в корне проекта:
- `testdata/photo1.jpg`
- `testdata/photo2.png`
- `testdata/photo3.webp`

При необходимости вы можете заменить их своими файлами (сохранив имена).

### Запуск всех тестов

```bash
go test -v ./...
```

**Обратите внимание:** Старый тест `myAdvertisement` может упасть, если у пользователя нет объявлений.

### Запуск только ваших тестов (создание объявлений)

```bash
go test -v ./tests/scenarios/advertisementsCreate -run TestAdvertisementsCreateSuite
```

Будут выполнены:
- **Позитивные тесты**: создание, получение по ID, проверка фото, поиск.
- **Негативные тесты**: без авторизации, отсутствие обязательных полей, некорректные значения.

### Логирование

Для вывода деталей HTTP-запросов и ответов установите в `.env.override`:

```env
DEBUG=true
```

### Запуск линтера

Установите [golangci-lint](https://golangci-lint.run/docs/welcome/install/local/) и выполните в корне:

```bash
golangci-lint run
```

Все замечания должны быть исправлены.

## Примечание для проверяющего

Папка `testdata/` с фото включил в репозиторий намеренно, подумал, что так проверка пройдет быстрее).  
Файл `.env.override` с реальными данными **не включён** в репозиторий, но в примере выше указаны существующие данные, можно использовать их.
